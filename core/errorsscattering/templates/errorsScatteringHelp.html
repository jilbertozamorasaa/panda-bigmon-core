The page represents a correlation matrix between ATLAS production Requests/Tasks and Regions/Queues.

The displayed tasks are in running or submitting states only. Paused tasks are removed.

On the main page the first column shows all the active production Requests and the first row shows all the Regions participating in production. The second column displays the number of Tasks in each Request with various job execution success rates (>80%, <80%, <50%). The second row displays the number of Queues in each Region with various job execution success rates (>80%, <80%, <50%). The cells of the table show the production job success rate, as well as number of finished and failed jobs for each Request vs. Region combination/intersection.

Legend on the upper right corner clarifies the color codding of the cells and numbers used in the page. Lighter colors indicate cells with low job statistics (number of jobs < 10).

Rows are ordered by average success rate, with Requests having the highest failing rate on top.

Each Request (first column) has a sign in front. Clicking it will open a similar correlation matrix page with all the Tasks belonging to that Request. Clicking on the Request number on the other hand will open the panda page of that Request.

Each Region (first row) has a sign under the Region name. Clicking it will open a similar correlation matrix page with all the Queues belonging to that Region. Clicking on the Region name on the other hand will open the panda page of that Region.

The default time window for the overall job statistics at each cell is 8 hours. But the page has a Time window menu with other time window options (24 hours, 7 days). You can also manually modify the time window to any other value in the page URL.

The page has various display or filtering options through the following buttons:

  • Switch to the queues view button allows you to display all the queues at once.
  • Exclude scouts button will exclude the scouting jobs from the statistics, so basically the matrix will skip the Requests/Tasks which are still running scouting jobs.
  • Hide low statistics cells button will omit the cells with <100 statistics, so basically it will condense the matrix, showing only the cells with higher contributions to production.
